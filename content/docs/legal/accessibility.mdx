# Accessibility Statement

**Last Updated:** March 2026

SmartFall is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying relevant accessibility standards.

## Accessibility Features

SmartFall incorporates the following accessibility features:

### Visual Accessibility

- **Color Contrast:** Text meets WCAG AA standards (4.5:1 contrast ratio)
- **Resizable Text:** All text can be scaled up to 200% without loss of functionality
- **Dark Mode:** Full dark theme support for reduced eye strain
- **Large Font Options:** Available through device/browser settings

### Keyboard Navigation

- **Tab Navigation:** All interactive elements are keyboard accessible
- **Keyboard Shortcuts:** Platform-standard shortcuts supported
- **Skip Links:** Skip to main content functionality
- **Focus Indicators:** Clear focus indicators on all interactive elements

### Screen Reader Support

- **ARIA Labels:** Proper semantic HTML and ARIA attributes
- **Alt Text:** Descriptive alt text for all images
- **Headings Structure:** Logical heading hierarchy for content
- **Form Labels:** All forms properly labeled for screen readers

### Audio/Video

- **Captions:** Video content includes captions
- **Transcripts:** Audio content provides transcripts
- **Audio Descriptions:** Complex visual content has audio descriptions

### Motor Control

- **Large Touch Targets:** Buttons and interactive elements are appropriately sized
- **Device Support:** Compatible with assistive input devices
- **No Hover Traps:** Information not hidden behind hover-only states
- **Sufficient Time:** No automatic redirects or time limits

## Accessibility Standards

SmartFall aims to conform to:

- **WCAG 2.1 Level AA:** Web Content Accessibility Guidelines
- **Section 508:** U.S. Federal accessibility requirements
- **ADA:** Americans with Disabilities Act compliance

## Known Limitations

We are aware of the following accessibility limitations and are working to address them:

- Some third-party software components may have limited accessibility
- PDFs generated by the system may not be fully accessible
- Real-time data visualizations may require alternative representations

## Ongoing Improvements

We continually work to improve accessibility by:

- Conducting regular accessibility audits
- Gathering feedback from users with disabilities
- Training staff on accessibility best practices
- Testing with assistive technologies
- Staying current with accessibility standards

## Getting Help

If you experience accessibility barriers:

### Report Issues

- **Email:** accessibility@smartfall.app
- **Phone:** 1-800-FALL-911
- **In-App:** Use the Accessibility Feedback button

### Provide Feedback

We welcome suggestions for improving accessibility. Please share:

- What barrier you encountered
- What device/browser you used
- How it affected your ability to use the service
- Your preferred solution

### Alternative Access

If you encounter barriers preventing service use:

- We can provide alternative formats for information
- Phone support is available for account/data access
- Emergency alerts can be configured for accessibility needs

## Contact Information

For accessibility questions or concerns:

- **Email:** accessibility@smartfall.app
- **Support Phone:** 1-800-FALL-911 (with TTY relay available)
- **Response Time:** We aim to respond within 24-48 business hours

## Policy History

This accessibility statement is regularly reviewed and updated to reflect our ongoing commitment to accessibility and inclusion.
